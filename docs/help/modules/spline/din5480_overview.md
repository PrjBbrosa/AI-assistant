# 渐开线花键齿面承压校核（DIN 5480 / DIN 6892 风格）

## 一图总览

```
        外花键（轴）                 内花键（轮毂）
     ┌──────────────┐             ┌──────────────┐
     │   d_a1       │             │   d_a2       │
     │  ┌────┐      │   啮合      │      ┌────┐  │
     │  │    │──────┼─────────────┼──────│    │  │
     │  │    │  z 齿、m 模数、L 长  │    │  │
     │  └────┘      │             │      └────┘  │
     │   d_f1       │             │              │
     └──────────────┘             └──────────────┘

 齿面平均承压：p = 2·T·k_α / (z · h_w · d_m · L)
 其中 h_w = (d_a1 - d_a2) / 2    [有效齿高，单侧]
      d_m = (d_a1 + d_a2) / 2    [接触区平均直径]
      T:  N·mm（已包含 K_A）
      p:  MPa
```

## 解决什么问题

给定渐开线花键的几何（模数 m、齿数 z、各圆直径、啮合长度 L）、载荷（设计扭矩 T、工况系数 K_A、载荷分布系数 k_α）与许用齿面压力 p_zul，校核**齿面平均承压是否低于许用值**，并给出安全系数 SF = p_zul / p_flank。

同时可选择"联合模式"：对花键轴的光滑过盈段同时走 DIN 7190 圆柱过盈配合校核，给出防滑、材料应力两个独立安全系数。

> Cannot verify against original DIN standard —— DIN 5480（渐开线花键尺寸与公差）与 DIN 6892（花键轴-轮毂连接承载能力）原文未逐条比对。本工具只是"DIN 6892 风格的工程简化估算器"，不替代完整标准合规校核。

## 核心流程（5 步）

1. **几何输入**：选择 DIN 5480 标准规格（下拉查表），或手动输入 m/z/d/d_a1/d_a2/d_f1；两种模式下都要给啮合长度 L。
2. **派生 h_w 与 d_m**：`h_w = (d_a1 - d_a2)/2`、`d_m = (d_a1 + d_a2)/2`。若使用近似推导（模数 + 齿数），工具会以 DIN 5480-2 catalog 最小 h_w/m = 0.5 作保守下限（详见"本工具实现"段）。
3. **设计扭矩预乘 K_A**：`T_design = T_required · K_A`。K_A 在工具入口一次性预乘，`_calculate_scenario_a` 内部的 `p_flank` 公式里只显式乘 k_α。
4. **齿面压力 p_flank**：`p_flank = 2·T_design·k_α / (z·h_w·d_m·L)`，单位换算 T: N·m → N·mm 时乘 1000。
5. **比对 p_zul 并给安全系数**：`SF = p_zul / p_flank`；同时给扭矩视角的 `T_cap = p_zul·z·h_w·d_m·L / (2·k_α)` 以及 `T_cap/T_design`（与 SF 数学等价）。

## 本工具实现的范围

**已实现**：
- 齿面平均承压简化校核（DIN 6892 风格），见 `core/spline/calculator.py:_calculate_scenario_a`。
- DIN 5480 内置 catalog（W 15–50 常用规格，30° 压力角系列），选择规格后自动填充 d/d_a1/d_a2/d_f1/m/z。
- 公开 / 图纸尺寸模式：要求用户显式提供 d_a1 / d_a2 / d_f1 四个尺寸。
- 近似模式：在仅有模数 m 与齿数 z 时，**使用 DIN 5480-2:2015 catalog 最小 h_w/m = 0.5 作保守下限** → `d_a1 = d - 0.5·m`、`d_a2 = d - 1.5·m`、`d_f1 = d - 2.0·m`。不同规格的实际 h_w/m 在 0.5~1.08 范围内波动，近似偏差方向始终让 p_flank 偏**高**（保守），避免假 PASS。
- 联合模式：花键 + 光滑过盈段同时校核；光滑段走 DIN 7190 Lamé 公式，设计扭矩已在入口预乘 K_A，委托调用里传 `ka = 1.0` 避免重复放大（见 `calculator.py:174` 注释）。
- 扭矩容量 `T_cap` 与扭矩视角安全系数 `torque_capacity_sf = T_cap / T_design`。**`torque_capacity_sf` 与齿面 `flank_safety` 数学等价**（都是 `p_zul / p_flank`），只是换算视角，不是两个独立判据。
- 结果级别 `overall_verdict_level = "simplified_precheck"`：UI 显示"PRECHECK PASS/FAIL"，提示用户这只是预校核。

**未实现**（显式列为 `not_covered_checks` 返回给 UI）：
- 齿根弯曲强度
- 剪切承载
- 内花键胀裂 / 轮毂局部强度
- 磨损与寿命
- 完整公差 / 变位 / 齿侧间隙链
- DIN 6892 K_1 · K_2 · K_3 系数族的逐一分解（本工具把三者合成为单一 `k_alpha` 保守上限，默认 1.3）
- 30° 以外的 DIN 5480 压力角变体（catalog 只覆盖 30° 系列）

## 常见误用

- **把扭矩容量 `T_cap/T_design` 当作独立于齿面 SF 的安全判据**：二者是同一个 `p_zul / p_flank`，只是表述角度不同。工具同时给出只是方便用户从"扭矩够不够"视角读结果。
- **对近似模式的预校核结果用作设计依据**：近似模式 h_w/m = 0.5 是保守下限，对 m ≥ 1.75 规格会把 p_flank 算得明显偏高。正式设计必须切到公开 / 图纸尺寸模式并录入实测值（UI 切换后会把几何字段锁为 `AutoCalcCard` 样式）。
- **k_alpha 当成 K_A**：K_A 在工况选项里（考虑驱动 / 负载冲击），k_alpha 在花键几何里（考虑齿向 / 齿面载荷分布不均）。两者相乘进入齿面压力公式，漏一个安全系数会被虚高至少 20–50%。
- **以为"simplified_precheck PASS"等同于 DIN 6892 合规**：`overall_verdict_level` 值被明确设为 `simplified_precheck`，不是 `din_6892_compliant`。若需要标准合规性结论，必须用完整 DIN 6892 计算工具或委托第三方校核。
