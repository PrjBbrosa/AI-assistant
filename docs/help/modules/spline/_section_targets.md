# 校核目标章节

本章节用于**确定校核模式和安全系数门槛**：选择仅做花键齿面预校核，还是同时校核光滑段过盈；并给出三类最小安全系数（齿面、防滑、材料应力）以及统一的工况系数 K_A。

## 为什么要填这些

花键连接的实际形态差异大。有些场合只关心花键齿面会不会被压溃（场景 A），几何上是"纯齿啮合 + 另外的键 / 轴承定位"；另一些场合花键轴上除了齿段还有一段光滑圆柱压入轮毂孔（场景 B），需要同时防滑、防塑性屈服。两种模式要用的字段差别很大，因此在第一步就决定模式，避免后面章节填了一半才发现不相关。

此外，各安全系数门槛直接决定 PASS / FAIL 判据，而 K_A 在计算入口就乘到设计扭矩上，对场景 A 的齿面压力和场景 B 的面压同时放大。

## 输入 / 产出

**输入**：
- **校核模式**：`仅花键` / `联合`。`仅花键`时光滑段章节的字段会被锁为 `AutoCalcCard` 禁用态，不参与 payload 构建（见 `spline_fit_page.py:_build_payload` 的 `active_sections` 逻辑）；`联合`时同时启用光滑段相关校核阈值（防滑 / 应力）。
- **齿面最小安全系数 S_flank,min**：场景 A 比对 `flank_safety = p_zul / p_flank` 的门槛。默认 1.30。
- **防滑最小安全系数 S_slip,min**：场景 B 光滑段防滑校核的门槛，默认 1.50。**`仅花键`模式下此字段被锁定**，不进入 payload。
- **材料最小安全系数 S_sigma,min**：场景 B 轴 / 轮毂应力校核门槛，默认 1.20。**`仅花键`模式下此字段被锁定**。
- **工况系数 K_A**：放大设计扭矩，考虑驱动 / 负载的冲击与过载特性。电机约 1.0~1.25，内燃机约 1.25~1.75。

**产出**：
- `T_design = T_required · K_A`（在 `calculate_spline_fit` 入口一次性预乘），之后的场景 A、场景 B 都基于这个放大后的扭矩，避免重复计入。
- 场景 B 委托调用 `calculate_interference_fit` 时会明确传 `ka = 1.0`（payload 里 `delegated_application_factor_ka = 1.0`），防止 DIN 7190 内部再次放大。

## 模式差异（本工具实际行为）

| 模式 | 场景 A 齿面承压 | 场景 B 光滑段防滑 / 应力 | `overall_verdict_level` |
|---|---|---|---|
| `仅花键` | 执行 | **跳过** | `simplified_precheck` |
| `联合` | 执行 | 执行 | `simplified_precheck` |

**不论哪种模式，`overall_verdict_level` 始终为 `simplified_precheck`**。UI 显示 "PRECHECK PASS / FAIL"，意在提示用户：场景 A 只是"齿面平均承压"的预校核，不包含齿根弯曲、剪切、胀裂、磨损、寿命、完整公差链；场景 B 虽然走 DIN 7190 Lamé 解析解，但也只校核防滑与屈服，不替代完整的 DIN 6892 花键连接承载能力评估。

## 常见坑

- **K_A 误填为 k_alpha**：两个都是"载荷放大系数"，但含义不同。K_A 是外部工况（机器类别），k_alpha 是花键内部载荷不均（K_1·K_2·K_3 合成）。分开填、分别调整。
- **切到`仅花键`后忘记光滑段字段仍然有默认值**：UI 会把 `smooth_*` 字段锁为 `AutoCalcCard` 禁用态，但字段值保留。这不影响计算（`_build_payload` 不把这些 section 放进 `active_sections`），只是视觉上仍可见，避免切回`联合`时又要重新填。
- **最小安全系数取过高**：齿面 1.5 通常就偏保守；≥ 2.0 多见于高冲击 / 非金属材料 / 长期连续运转场合，不要盲目套。
