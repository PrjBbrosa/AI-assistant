# 基本设置章节

本章节用于**设置校核范围和计算方法**：决定是否启用齿面/齿根负载能力校核，以及采用 DIN 3996 的哪一种方法。

## 为什么要填这些

蜗轮校核分两层：几何校核（必做，DIN 3975）确认尺寸、啮合、干涉是否自洽；负载能力校核（可选，DIN 3996）计算齿面接触应力 σH、齿根弯曲应力 σF 并与许用值对比。短寿命、低功率或仅作方案比较的传动，可以只做几何；减速机、电动推杆、长期连续运转的蜗轮必须开启负载能力校核。

这一步的选择直接决定后续表单的可见性：启用负载能力后，Load Capacity 章节才展开，许用应力、使用系数 KA、动载系数 Kv 等字段才变成必填。

## 输入 / 产出

**输入**：
- 项目备注：用于报告封面标识，不参与数值计算
- 是否启用负载能力校核
- DIN 3996 方法选择（A / B / C）

**产出**：
- 决定 Load Capacity 章节是否显示
- 决定主流程分支（纯几何 vs 几何 + 负载能力）
- 在计算报告封面标注本次校核采用的方法编号

## 方法差异（本模块实际行为）

DIN 3996 定义了三种计算详细度。本模块的实际落地情况如下：

- **Method B**（默认，推荐）：解析简化法，用单点 Hertz 线接触应力 + 悬臂梁齿根弯曲近似，配合经验系数 KA/Kv/KHα/KHβ。工程实际最常用，也是本模块的主路径。
- **Method A**：在 Method B 结果的基础上叠加手册系数法折减 —— 效率 × 0.92、齿面接触应力 × 0.95，用于装备选型时给出更保守的估值。**实测会改变数值输出**，不是仅贴标签。
- **Method C**：需要 FEA 输入或沿接触线逐段数值积分，**本模块未实现**，选中后 calculator 会直接抛 `InputError: Method C 需要 FEA 输入，当前版本未实现`。

> 注：本模块不是完整 DIN 3996 实现，只对 Method B 做了"单点解析 + 线接触 Hertz 近似 + 悬臂梁齿根近似"的简化落地。Cannot verify against original DIN standard —— 原始 DIN 3996:2019 条款尚未逐项比对。正式设计评审请以原始标准为准。
