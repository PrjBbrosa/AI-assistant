# 接触模型与几何章节

本章节用于**选择接触类型（线接触 / 点接触）并填写两接触体的曲率半径及接触长度**。几何直接决定等效曲率半径 R' 和接触斑形状，是赫兹应力计算的核心输入。

## 为什么要填这些

赫兹接触应力的基本前提是"两个弹性体在一个几乎平面的小接触区相互压触"。这个小接触区的形状和大小由两体的主曲率半径决定：

- **线接触**：两圆柱轴线平行（或圆柱-平面）→ 接触区为细长矩形，按单位长度载荷 `q = F / L` 计算；接触斑半宽 `b = √(4·q·R' / (π·E'))` [q: N/mm, R': mm, E': MPa → b: mm]
- **点接触**：两球相切，或球-平面（各向同性 R1 = R2 → 简化等轴对称） → 接触区为圆，按法向载荷 F 计算；接触斑半径 `a = ((3·F·R') / (4·E'))^(1/3)` [F: N, R': mm, E': MPa → a: mm]

两种模型对同一组 (R', E', F) 给出**完全不同**的 p0 值（同等载荷下点接触 p0 显著高于线接触，因为接触区更集中）。**模式选错会直接把结果换一个量级**。

## 输入 / 产出

**输入**：
- **接触类型**：线接触 / 点接触（必选）
- **曲率半径 R1 [mm]**：第 1 接触体主曲率半径；凸面取正值；平面填 0
- **曲率半径 R2 [mm]**：第 2 接触体主曲率半径；平面填 0
- **接触长度 L [mm]**：仅线接触需要；点接触时该字段禁用（UI 置灰），值不参与计算

**产出**：
- `derived.r_eq_mm = R'`：等效曲率半径，`1/R' = 1/R1 + 1/R2` [mm]
- `contact.semi_width_mm = b`（线接触）或 `contact.contact_radius_mm = a`（点接触）
- `contact.contact_area_mm2`：线接触 `A = 2·b·L`；点接触 `A = π·a²`

## 方法差异

- **线 vs 点**：本工具按用户选择的接触模式走不同公式分支，结果字段名不同（`semi_width_mm` vs `contact_radius_mm`）；报告与 UI 根据 `result["mode"]` 字段自动适配显示。
- **平面用 0 表示**：`R = 0` 在内部被映射为 `1/R = 0`（无限大半径）。代码在 `_radius_inverse` 中显式实现，请勿填入极大数（如 1e9）代替平面——结果一致，但语义不清。
- **凹面（内滚道、蜗轮齿沟）未支持负值**：`_positive(..., allow_zero=True)` 拒绝负数。若需要凹凸配对（如滚动轴承外圈内滚道），应先离线计算等效曲率 R' 后反推"等效凸 R1"作为输入，并在"接触长度"/"许用值"里记录原工况的关键假设。

## 输入 / 产出单位对照

| 字段          | UI 单位 | 内部使用 | 备注                  |
| ------------- | ------- | -------- | --------------------- |
| R1 / R2       | mm      | mm       | 0 表示平面            |
| L（线接触）   | mm      | mm       | 点接触时不参与        |
| R'（派生）    | mm      | mm       | 1/R' = 1/R1 + 1/R2    |
| b / a（派生） | mm      | mm       | 线：半宽；点：半径    |

## 常见坑

- **点接触模式下仍然改动"接触长度 L"期望影响结果**：UI 会把 L 置灰并显示为只读，但若用户绕过 UI 通过保存/加载的 JSON 改动该值，`calculator` 只在 `mode == "line"` 时读 L，点接触完全忽略。
- **两个平面互相接触（R1 = R2 = 0）**：`inv_r = 0` → 抛 `InputError: 等效曲率为 0`。赫兹接触要求至少一侧有曲率。
- **把直径当成半径填入**：R1、R2 是**半径**，不是直径；滚动体是直径 φ20 的钢球时填 10。
- **球-球外切 vs 球-球内切**：外切两球都填正值 R1、R2；内切（小球在大球凹面内）当前不支持，需按凹面配对的等效方法外部处理。
