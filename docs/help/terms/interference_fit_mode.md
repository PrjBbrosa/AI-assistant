# 过盈来源模式（fit.mode）

**一句话**：选择过盈量窗口的输入来源——直接填写过盈量 / 选择优选配合代号 / 按轴孔偏差换算；三种模式给出的 (δ_min, δ_max) 会被后续所有校核统一使用。

**怎么理解**：

工程上描述过盈配合的三种常见语境：

1. **直接给过盈量**：已有上一轮工艺计算结果 / 竞品拆解数据，手上已有目标 δ_min / δ_max。
2. **配合代号**：图纸标注 "Ø40 H7/s6" 这类代号，系统按 ISO 286 查表反推过盈量。
3. **轴孔偏差**：自制件或非标件，直接给出轴上下偏差、孔上下偏差，计算过盈窗口。

本工具用 `fit.mode` 字段切换三种输入模式，保证内部后续流程**统一消费 (δ_min, δ_max)** 两个数字。

## 三种模式（本工具实现）

### 模式 1：直接输入过盈量

```
UI 字段激活:    fit.delta_min_um、fit.delta_max_um
UI 字段锁定:    fit.preferred_fit_name、fit.shaft_*_deviation_um、fit.hub_*_deviation_um
校核数据来源:   直接使用用户输入
```

适合：已有现成过盈目标、快速试算、教学案例。

### 模式 2：优选配合代号

```
UI 字段激活:    fit.preferred_fit_name（下拉选 H7/p6、H7/s6、H7/u6）
UI 字段锁定:    fit.delta_* 由程序自动填入并灰色显示；偏差字段锁定
数据查表逻辑:   按配合代号 + 公称直径 d 查 ISO 286 子集表
                → 自动换算 δ_min = shaft_lower − hole_upper
                   δ_max = shaft_upper − hole_lower
```

**本工具内置表限制**：
- 代号：仅 `H7/p6`、`H7/s6`、`H7/u6`
- 直径范围：6 ~ 50 mm
- 超出范围时：UI 会把 (δ_min, δ_max) 清零，需切到其他模式

### 模式 3：偏差换算

```
UI 字段激活:    fit.shaft_upper_deviation_um (es)
               fit.shaft_lower_deviation_um (ei)
               fit.hub_upper_deviation_um (ES)
               fit.hub_lower_deviation_um (EI)
UI 字段锁定:    fit.delta_* 由程序自动填入；fit.preferred_fit_name 锁定
换算公式:
   δ_min = ei − ES                              [um, 直径方向]
   δ_max = es − EI                              [um]

校验规则:
   es ≥ ei                                      轴偏差顺序正确
   ES ≥ EI                                      孔偏差顺序正确
   δ_max > 0                                    必须形成过盈
   δ_min ≥ 0                                    不是过渡 / 间隙配合
```

## 模式切换的 UI 联动

切换 `fit.mode` 时，UI 会自动：
1. **锁定不相关的字段卡**（变成蓝灰色 AutoCalcCard 风格）
2. **自动填入 δ_min, δ_max 到只读字段**（模式 2、3 下）
3. **清除旧的衍生值**（避免切回模式 1 时残留其他模式的数据）

## 三种模式各自的坑

**模式 1（直接输入）**：
- 容易把径向过盈填进直径字段（差 2 倍）
- 没有公差追溯；报告里只有两个数字，缺少来源依据

**模式 2（优选配合）**：
- 超范围时（d < 6 mm 或 d > 50 mm）直接 fallback 到 (0, 0)；**必须**切模式手填
- 只内置三种代号；其他配合（如 P7/h6、N6/h5）需要自行换算

**模式 3（偏差换算）**：
- 轴孔偏差符号容易填反：孔 ES / EI 常为负值（孔比名义小），轴 es / ei 常为正值（轴比名义大）
- δ_min < 0 时代表当前是过渡 / 间隙配合；本工具直接报错，不默默续算

## 常见误用

- **已切到模式 2 仍手动改 δ_min**：字段虽锁定但可能通过 JSON 载入绕开；以 UI 模式切换为准。
- **偏差换算后发现 δ_min 很小**：提示此配合对**最小过盈工况**不够稳定，应换更重过盈代号。
- **混用径向 / 直径单位**：本工具要求直径方向 um；径向数据应乘以 2。
- **JSON 载入旧文件时 `fit_selection` 缺失**：UI 默认 fallback 到 "直接输入过盈量" 模式；检查 δ_min, δ_max 是否合理。

## 出处

- ISO 286-1 / ISO 286-2 公差与配合（优选配合查表）
- DIN 7190-1 对过盈量窗口的定义

> Cannot verify against original DIN standard —— 本工具内置的优选配合表是 ISO 286 的**子集**，覆盖范围有限；**未查证 DIN 7190-1 / ISO 286 原文**的完整内容，故不给出精确节号 / 表号。
