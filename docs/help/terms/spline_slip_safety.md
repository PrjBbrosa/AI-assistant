# 防滑最小安全系数（S_slip,min）

**一句话**：`联合`模式下场景 B 光滑段的防滑校核门槛；要求 `torque_sf = T_cap_slip / T_design ≥ S_slip,min` 才判 PASS。**仅在`联合`模式下参与计算**，`仅花键`模式下此字段被锁为 `AutoCalcCard` 禁用态。

**怎么理解**：

花键轴光滑段压入轮毂孔后，产生径向接触压力 p；摩擦系数 μ_torque 下，这段光滑配合能抵抗的最大扭矩为：

```
T_cap_slip = p_min · A_contact · μ_torque · (d / 2)    [N·mm]
                                                  [p: MPa, A: mm², d: mm → T: N·mm]
```

其中 `p_min` 取**最小过盈量 δ_min 对应的**接触压力（防滑看下限）；`A_contact = π · d · L_fit` 是有效配合面积。

防滑安全系数：
```
torque_sf = T_cap_slip / T_design   [无量纲]
PASS 条件： torque_sf >= S_slip,min
```

场景 B 的判据在 `core/interference/calculator.py` 里实现，花键模块通过 `_calculate_scenario_b` 委托调用。

**为什么要取最小过盈 δ_min 而不是名义或最大**：

过盈量有公差（`δ_min` 与 `δ_max`），实际装配出来的过盈值可能落在任何一点。**防滑校核必须考虑最坏情况 —— 最小过盈**，因为这决定接触压力的下限、决定摩擦可抵抗扭矩的下限。反之，**屈服校核要用 `δ_max`**（最大过盈对应最高应力），见 `spline_stress_safety`。两个校核分别取两端，不能统一用中间值。

**典型值 / 场景**：

- **1.3 ~ 1.5**：常规工业过盈（中等摩擦系数、稳定载荷）。本工具默认 1.50。
- **1.5 ~ 2.0**：摩擦系数取值不确定（加工残留油、表面氧化、装配过程微滑）、或载荷有脉动。
- **2.0 ~ 3.0**：摩擦环境恶劣（低温、高温润滑失效、振动脱贴合）或后果严重（飞轮 / 联轴器脱开威胁人员安全）。
- **≥ 3.0**：极端场合或要求按 DIN 7190 保守侧加倍取值。

**常见误用**：

- **把 `仅花键`模式下的 S_slip,min 默认值误当作生效**：`仅花键`模式下字段被锁定，不会进入 payload；切回`联合`时才有效。
- **误以为摩擦系数高 → 可以把 S_slip,min 压低**：μ_torque 的取值本身已经是保守侧（0.10~0.14），继续压低安全系数把两个"宽松"叠加，实际裕度会不足。
- **在光滑段 + 花键齿段同时传扭矩的场合重复计算**：本工具的场景 A 与场景 B 都用同一个 `T_design` 做独立校核，是"两个都要通过"的串联逻辑，不是"两段分担"。若实际结构确实按承载比例分担，需要在外部先拆分再填两套计算。

**出处**：DIN 7190-1:2017 给出了过盈配合防滑的解析公式；S_slip,min 取值基于工程经验。
