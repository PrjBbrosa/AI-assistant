# 几何输入模式（公开/图纸尺寸 vs 近似推导）

**一句话**：决定场景 A 的花键几何是按**用户提供的实测 / 图纸 / catalog 尺寸**来算，还是按**保守下限近似**来算。近似模式的 p_flank 永远不低于实际值，只适合方案阶段快速判断。

**怎么理解**：

花键齿面压力公式里的 h_w（有效齿高）和 d_m（平均直径）都由 d_a1（外花键齿顶圆）、d_a2（内花键齿顶圆）派生：

```
h_w = (d_a1 - d_a2) / 2
d_m = (d_a1 + d_a2) / 2
```

但 d_a1 / d_a2 在 DIN 5480 的 catalog 里是跟具体规格挂钩的 —— 不同 m × z 组合下，h_w / m 的比值在 0.5 ~ 1.08 之间显著波动。如果只有模数 m 和齿数 z（没有查 catalog），工具要不要估一个 h_w？这就是两种模式的分歧。

**模式 1：公开/图纸尺寸（`reference_dimensions`，推荐）**

要求用户**显式提供四个尺寸**：`reference_diameter_mm`、`tip_diameter_shaft_mm` (d_a1)、`root_diameter_shaft_mm` (d_f1)、`tip_diameter_hub_mm` (d_a2)。缺一个就抛 `GeometryError: 显式花键几何输入不完整...`。

数据来源（按可靠性从高到低）：
1. 实测（装配前用千分尺或投影仪测量）
2. 零件图标注尺寸（带公差）
3. DIN 5480-2 catalog 条目（选"标准花键规格"下拉，自动填入）

工具会额外校验 `d_f1 < d_a2 < d_a1 < d` 的物理顺序；若 m × z 与 d 相对偏差 > 5% 会追加警告（不阻断）。

**模式 2：近似推导（`approximate`，仅预估）**

只给模数 m 与齿数 z，工具按以下**保守下限**派生：
```
d    = m · z                 [参考直径]
d_a1 = d - 0.5 · m           [外花键齿顶]
d_a2 = d - 1.5 · m           [内花键齿顶]
d_f1 = d - 2.0 · m           [外花键齿根]
h_w  = 0.5 · m               [DIN 5480-2 catalog 最小值]
d_m  = d - 1.0 · m
```

**为什么用 h_w / m = 0.5 作下限**：`core/spline/geometry.py:derive_involute_geometry` 的注释解释得很清楚 —— DIN 5480-2:2015 catalog 中不同条目的 h_w/m 最小值 ≈ 0.5（出现在小齿数、大模数的极端规格如 W 25×2.5×8 / W 20×2×8）。取这个下限让 `p_flank = 2T·k_α/(z·h_w·d_m·L)` 始终**≥ 真实值**（h_w 小 → p_flank 大 → SF 小 → 更难 PASS）。这样即便真实 h_w 是 1.0·m，近似模式给出的结果也只是"更悲观"，不会"假 PASS"。

**两模式的数值差异**：

同一工况下，近似模式的 `p_flank` 相对于"真实 h_w = 0.9·m 条目"会**高 80%**（因为 0.9/0.5 ≈ 1.8）。这意味着：
- 近似模式下 PASS → 真实几何下大概率也 PASS（结论可信）。
- 近似模式下 FAIL → 真实几何可能 PASS 也可能 FAIL（需要切到`公开/图纸尺寸`再判断）。

**典型值 / 场景**：

- **方案选型阶段**：只知道"大概用 m=1.25、z=10 的花键"，还没定具体 catalog 条目。选`近似推导`快速判断承载是否在量级上足够。
- **零件图已出**：d_a1 / d_a2 / d_f1 都有明确标注，切`公开/图纸尺寸`并录入实测值。
- **选了标准 catalog 规格**：UI 自动切到`公开/图纸尺寸`模式并把 catalog 值填入锁定字段，用户不需要手动切换。

**常见误用**：

- **近似模式下 PASS 就认为设计 OK**：近似模式只是"保守下限判据"，不包含齿根弯曲、磨损、公差链等。详细设计阶段必须走`公开/图纸尺寸`。
- **手动录入 d_a1 / d_a2 但又切到`近似推导`**：工具会忽略显式尺寸、按 m × z 重新推 d_a1 / d_a2。这通常不是用户意图。
- **把 `approximation_used` 返回值当无害**：工具会在结果消息区追加"近似模式使用 DIN 5480-2 catalog 最小 h_w/m=0.5 作保守下限..."提示，提醒这是预估。

**出处**：DIN 5480-2:2015 catalog 数据范围；本工具实现细节见 `core/spline/geometry.py:derive_involute_geometry`。

> Cannot verify against original DIN standard —— h_w/m 的实际 catalog 最小 / 最大值未逐条与原表比对，本工具约定取 0.5 作下限基于公开样例整理。
