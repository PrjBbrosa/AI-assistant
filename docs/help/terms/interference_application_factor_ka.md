# 工况系数 KA（过盈配合）

**一句话**：把**名义载荷放大到设计载荷**的系数；过盈配合所有需求载荷（扭矩、轴向力、径向力、弯矩）都统一乘上它，再与容量比较。

**怎么理解**：

工程设计里"需求扭矩 T_req"一般是指**稳态传递扭矩**（额定工况下的代表值）。但真实运行总有波动：
- 启停冲击（电机启动、刹车、碰撞）
- 负载脉动（往复机、齿轮啮合、皮带打滑）
- 外部冲击（工件卡死、刀具碰撞）

这些瞬时峰值可能是额定值的 1.5~3 倍甚至更高。过盈连接若只按 T_req 校核，遇到真实工况就会滑脱。KA 的作用就是在进入校核之前**把名义载荷放大到最坏瞬时工况**。

## 重要：过盈配合的 KA ≠ 齿轮的 KA

KA 在机械设计里是常见符号，但不同标准的语义差别大：

| 标准 / 场景 | KA 的含义 |
|---|---|
| **DIN 7190 过盈配合**（本工具） | 把名义扭矩放大到设计扭矩的单一系数，按工况类型查表 |
| DIN 3990 / ISO 6336 齿轮 | 齿轮 KA 查表依据不同齿型族（原动机 + 从动机组合） |
| VDI 2230 螺栓连接 | 没有直接对应的"KA"，更关注拧紧系数 αA |

**术语命名前缀**：本术语叫 `interference_application_factor_ka`，与齿轮的 `gear_application_factor_ka` 分开；查表背景、典型数值、工程解释都与齿轮族不同，**不要混用**。

## 公式（本工具实现）

```
设计载荷 = KA × 需求载荷:
   T_design   = KA · T_req          [T: N·m]
   F_design   = KA · F_req          [F: N]
   F_r,design = KA · F_r,req        [F: N]
   M_b,design = KA · M_b,req        [M: N·m]
```

KA 对**所有**载荷统一放大；不能只对扭矩乘 KA 不对径向力乘。本工具默认 `KA = 1.20`，最小允许值 > 0。

## 典型值（经验）

| 工况描述 | KA 建议 |
|---|---|
| **电动机稳态驱动均匀负载**（泵、风机、一般传动） | 1.0 ~ 1.25 |
| **中等冲击** / 偶有脉动（一般机床、输送机） | 1.25 ~ 1.5 |
| **明显冲击 / 启停频繁**（冲压机、压力机进给轴） | 1.5 ~ 2.0 |
| **严重冲击**（破碎机、锻造设备） | 2.0 ~ 2.25+ |
| **内燃机 / 往复压缩机驱动** | 1.25 ~ 1.75 |
| **电机短路 / 制动瞬态**（按瞬态 T 最大值） | 2.0 ~ 3.0 |

**说明**：上述数值是过盈连接工程实践常用取值，**不等同**于 DIN 3990 齿轮 KA 查表。精确数值应结合测功、仿真或试验校准。

## 工程选择建议

1. **不确定就取高一档**：KA 偏高让需求过盈量也偏高，偏保守。常见错误是"为了让校核通过调小 KA"，等于掩盖真实工况。
2. **已知瞬态 T 峰值时可直接用**：`KA = T_peak / T_rated`，不再需要查表。
3. **与 S_slip 区分清晰**：KA 是需求侧放大（按工况类型），S_slip,min 是容量侧门槛（按后果严重度）。两者分工，不可替代。
4. **极端冲击建议走疲劳校核**：KA 只做静态放大；循环疲劳场合（每分钟多次冲击）应启用 Step 5 fretting 评估并考虑更深入的疲劳分析。

## 常见误用

- **"用 KA 顶安全系数"**：KA = 1.5、S_slip,min = 1.0 看似放大了 1.5 倍，但工程评审会不认。KA 和 S_slip 分别对应两个不同语义，不是"乘法互换"。
- **径向力不乘 KA**：容易漏；本工具内部统一乘 KA，UI 输入保持名义值即可。
- **套用齿轮 KA 表**：齿轮 KA 查表依据原动机 + 从动机的组合，过盈场景只需看**后果严重度**与**瞬态峰值比**。
- **KA 设到 3.0 以上**：通常意味着工况不是过盈连接的适用场景，应考虑加键 / 销或其他防滑机构。
