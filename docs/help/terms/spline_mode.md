# 校核模式（仅花键 / 联合）

**一句话**：决定本工具要不要在校核花键齿面承压的同时再做一次光滑过盈段的 DIN 7190 圆柱过盈校核。两者对应的输入字段差异很大，先选模式再填后续章节。

**怎么理解**：

花键轴有两种常见几何结构——**纯花键**（整段都是齿，靠齿面传扭矩 + 由键 / 轴承端部定位）、**花键 + 光滑过盈段**（齿段传扭矩，相邻的光滑圆柱段压入轮毂孔做定位 + 分担扭矩 / 轴向力）。两种结构的失效模式不同：纯花键只要防止齿面被压溃；组合结构除了齿面，还要同时防止光滑段打滑和轮毂胀裂。

本工具用"校核模式"这一个开关把两条计算路径彻底分开：

- `仅花键` → 只跑场景 A（齿面平均承压简化校核）。
- `联合` → 场景 A + 场景 B（光滑段 DIN 7190 Lamé 解析校核）。

**在 `core/spline/calculator.py:calculate_spline_fit` 的实现**：
```
mode = data.get("mode", "spline_only")
scenario_a = _calculate_scenario_a(...)       [始终执行]
if mode == "combined":
    scenario_b = _calculate_scenario_b(...)   [联合模式才执行]
```

UI 层进一步把两种模式的字段可见性分开：`仅花键`模式下光滑段所有字段锁为 `AutoCalcCard` 禁用态，防滑 / 应力最小安全系数也一并锁定；切回`联合`会自动解锁并保留上次编辑值。

**典型值 / 场景**：

- **`仅花键`** 典型场景：
  - 花键轴只传扭矩，轴向由独立推力轴承 / 锁紧螺母定位；花键段与轮毂之间是间隙配合（滑移花键、伸缩万向节花键）。
  - 变速器拨叉操作的可滑移花键（运转中可相对轴向移动，不做过盈）。
  - 方案阶段快速判断花键齿面承载是否足够，不关心轮毂结合段。
- **`联合`** 典型场景：
  - 减速机输出轴：前半段花键传扭矩、后半段光滑圆柱与联轴器法兰过盈压配。
  - 电机转子轴 ↔ 齿轮轮毂：花键驱动、相邻光滑段过盈锁定径向位置。
  - 需要既防滑又防胀裂的关键连接，必须双校核。

**常见误用**：

- **结构本该是`联合`却选了`仅花键`**：只得到"齿面 PASS"结论就批准，结果实际工况下光滑段打滑或轮毂胀裂。
- **结构是纯花键却选了`联合`**：字段解锁后填了一堆不相关的过盈数据，计算会 FAIL 在场景 B（因为光滑段根本不存在，相当于 L = 0 / δ = 0）。应该选`仅花键`。
- **切换模式后没注意 `overall_verdict_level`**：两种模式下 `overall_verdict_level` 都是 `simplified_precheck`（不是 `din_6892_compliant`）—— 即便`联合`模式两个场景都 PASS，也只能作为预校核结论。

**出处**：本工具实现约定，无直接标准对应；DIN 5480 / DIN 6892 / DIN 7190 在标准语境下分别独立校核，本工具把组合结构合并成一个流程。

> Cannot verify against original DIN standard —— 校核模式开关是本工具的 UI 约定，不是 DIN 标准术语。
