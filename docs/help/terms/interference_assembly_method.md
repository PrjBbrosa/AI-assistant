# 装配模式（manual_only / shrink_fit / force_fit）

**一句话**：过盈连接的装配工艺选择；影响压入力 / 装配温度 / 装配摩擦的估算，但**不改变 DIN 7190 核心通过/不通过结论**。

**怎么理解**：

同一副过盈连接（δ、d、L、材料都一样）可以用不同工艺装配。装配模式只改变：
1. **装配阶段怎么把轴装进去**：压入 vs 热装 vs 不区分
2. **装配力估算口径**：用哪个摩擦系数 μ
3. **是否校核热装温度上限**（只 shrink_fit）

不改变的事情：
- 服役阶段传递扭矩 / 抗轴向 / 张口缝 / 屈服校核（这些由过盈量本身决定）
- 主 overall_pass 结论

## 三种装配模式（本工具实现）

### `manual_only` — 仅压入力估算

```
F_press = μ_Assy · p · π · d · L                       [F: N, p: MPa, d,L: mm]
```

- 使用 friction 章节的 μ_Assy 做通用压入力估算
- 不区分具体工艺，适合**工艺待定阶段**或**新手快速试算**
- UI 会把本章节的其他字段（温度、间隙、力_fit 摩擦）锁定为灰色

**何时选**：项目早期尺寸敲定阶段；工艺由后续环节另定。

### `shrink_fit` — 热装

需要解决两个问题：
1. **把轮毂加热到多少度才装得进去**
2. **这个温度是否超出材料允许上限**

```
装配间隙 clearance_um（本工具实现）：
   diameter_rule:   clearance_um = d_shaft            [d_shaft: mm → clearance: μm]
                                   （每 1 mm 直径预留 1 μm 间隙，即 d / 1000 mm）
   direct_value:    clearance_um = 用户输入           [μm]

所需轮毂加热温度（本工具实现，对照 core/interference/assembly.py:116-124）：
   required_expansion_um = δ_max + clearance_um       [δ: μm, clearance: μm → 累计: μm]
   hub_growth_per_c      = α_h · d_shaft / 1000       [α_h: 10⁻⁶/°C, d: mm → μm/°C]
   T_hub_required = T_amb
                  + required_expansion_um / hub_growth_per_c
                  + (α_shaft / α_hub) · (T_shaft − T_amb)
                                       [最后一项为轴预冷修正，单位: °C]
```

其中 `T_shaft` 若不等于室温（例如同时用干冰冷却轴 + 加热轮毂），修正项避免低估所需轮毂温度。本模块 `assembly.py:123` 明确叠加此项；此前版本误删了轴预冷修正。

若 `T_hub_required > hub_temp_limit_c`（材料允许上限），本工具会发出警告。

**典型材料热装温度上限**：
- 调质钢：250 ~ 300°C（超过会回火、硬度下降）
- 渗碳淬火钢：150 ~ 180°C（硬度层怕过热）
- 铝合金：150°C（过时效、组织粗化）
- 球铁：300°C（相对宽容）

**何时选**：
- 大过盈连接（压入困难）
- 大直径（压入力超设备）
- 需要保持精度的精密轴承 / 主轴（压装会擦伤）

### `force_fit` — 压装

```
F_in  = μ_in  · p · π · d · L                         [F: N]
F_out = μ_out · p · π · d · L                         [F: N]
```

μ_in（压入摩擦）：动摩擦，典型 0.06 ~ 0.10
μ_out（压出摩擦）：表面已经压过一次，通常略低，典型 0.05 ~ 0.08

**何时选**：
- 小 ~ 中等过盈连接
- 工艺无加热设备（小作坊 / 修理场）
- 轻量件 / 维修场景

## 三种模式对比表

| 特征 | manual_only | shrink_fit | force_fit |
|---|---|---|---|
| 压入力估算 | μ_Assy 通用 | 几乎无摩擦 | μ_in / μ_out 分离 |
| 装配温度校核 | 无 | 有（T_hub） | 无 |
| 设备需求 | 无特定 | 加热设备 | 液压 / 机械压机 |
| 适用过盈大小 | 任意 | 大 | 小到中 |
| 精度保持 | — | 好 | 一般（可能擦伤） |
| 工艺成本 | — | 中高 | 低 |

## UI 字段锁定规则

切换装配模式时本工具自动锁定 / 解锁字段卡：

| 字段卡 | manual_only | shrink_fit | force_fit |
|---|---|---|---|
| 热装温度 / 间隙 / α 系数 | 锁 | **开** | 锁 |
| 轮毂允许最高装配温度 | 锁 | **开** | 锁 |
| μ_press_in / μ_press_out | 锁 | 锁 | **开** |

锁定的字段值不进入计算，但会被保留以便切回时恢复。

## 为什么装配模式不影响主结论

过盈连接的服役性能只取决于**最终装配后的接触状态**。不管是压装还是热装，最终 δ_eff、p、应力都是一样的。

装配模式只改变"装配阶段本身"的工艺细节：
- 压装：压入力是否超设备
- 热装：加热温度是否超材料上限
- 不指定：只给通用压入力估算

## 常见误用

- **JSON 载入旧文件时装配模式缺失**：本工具 fallback 到 manual_only，字段锁定；想走热装 / 压装需手动切。
- **shrink_fit 但没填线膨胀系数 α_h**：默认 11·10⁻⁶/°C（钢）；铝轮毂应改 23·10⁻⁶/°C。
- **force_fit 但用 μ_Assy**：会被 UI 的 μ_in 覆盖；实际压入力按 μ_in 算。
- **工艺已定仍用 manual_only**：不会出错，但 workflow 报告不够细致，工艺追溯缺失。
- **shrink_fit 温度超 300°C**：调质钢可能被意外回火；应减小 δ_max 或分段过盈，而不是硬加热。
