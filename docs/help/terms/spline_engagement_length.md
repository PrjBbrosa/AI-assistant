# 有效啮合长度（L）

**一句话**：花键齿面沿轴向真正传递载荷的那一段长度，不是零件图上标注的"花键段总长"。过填会虚高扭矩容量，应按图纸标注的啮合深度或实测值取。

**怎么理解**：

花键齿面压力公式里的 L：

```
p_flank = 2·T_design·k_α / (z · h_w · d_m · L)   [MPa]
```

L 在分母，越小 p_flank 越大。因此用**偏小**的 L 会让校核更保守、用**偏大**的 L 会让校核更冒进。

实际零件上，"花键段总长" ≠ "有效啮合长度"：
```
  ├─ 端部倒角 / 引导锥 (2~3 mm) ─┤
  ├─────────── 花键段总长 ───────────┤
                ├─ 有效啮合 L ─┤
                                  ├─ 退刀槽 (2~5 mm) ─┤
```

- 端部倒角 / 引导锥：为便于装配，花键端部常做 30° 或 45° 锥形引导段，齿高从 0 渐增到 h_w。这一段齿高不够，传递载荷能力低，通常**不计入 L**。
- 退刀槽：花键与相邻光滑段之间的让刀凹槽，齿完全消失。也**不计入 L**。
- 齿顶硬化 / 软化过渡区：齿全高但热处理不稳定的区域（约 1~2 mm），保守起见也可以不计。

**本工具实现**：

- 场景 A 直接用用户填入的 `spline.engagement_length_mm` 作为 L，不做进一步扣减。
- 场景 B（联合模式下的**光滑段**）有独立的 `smooth_fit.fit_length_mm`（名义配合长度）和 `smooth_fit.relief_groove_width_mm`（退刀槽），工具把名义长度扣减退刀槽后得到有效长度：`L_fit = L_nominal - relief_groove`（见 `calculator.py:146-155`）。花键段的 L 和光滑段的 L_fit 是**两个独立概念**，不要混用。

**典型值**：

- **L ≈ 0.5 · d_B ~ 1.0 · d_B**：常规工业传动的推荐范围。例如 W 25 规格取 L = 15~25 mm。
- **L < 0.5 · d_B**：短啮合连接，用于空间受限场合；单齿载荷高，对冲击敏感。
- **L > 1.0 · d_B**：长啮合；理论上 p_flank 变小但 **实际承载并不成比例提升**，因为长啮合段的载荷分布不均（k_α 需要取更大），两个效应互相抵消。实际工程上很少做 L > 1.2·d_B 的花键（回报递减）。

**常见误用**：

- **填"花键段总长"**：包含了端部倒角和退刀槽。真实有效 L 会偏低 3~8 mm，尤其对 L < 20 mm 的小花键影响显著。
- **两场景用同一个 L**：场景 A 的 L 指**花键齿啮合段**；场景 B 的 L_fit 指**光滑过盈段**（不同几何段）。两段位于轴向相邻位置，但独立处理。
- **L 过大以补偿 p_flank**：加倍 L 只把 p_flank 降一半，但装配公差、加工误差会让实际只有前 60~80% 长度真正承载（齿向倾斜、轴变形）。工具的 k_α 已经包含了"载荷分布不均"的保守修正，不要在 L 上二次补偿。

**出处**：DIN 6892:2012（花键连接承载能力，L_eff 约定）；工程经验。
