# 工况系数（K_A，花键连接）

**一句话**：花键连接中考虑驱动 / 负载特性冲击与过载的外部放大系数，用来把名义扭矩放大到设计扭矩：`T_design = T_required · K_A`。同时放大场景 A 和场景 B 的载荷。

**怎么理解**：

名义扭矩（铭牌、电机额定、传动比反推得到）是"平均 / 稳态"值；实际运转中由于启停、换挡、冲击、负载波动，瞬时扭矩会显著高于名义值。K_A 是一个**查表系数**，把这种冲击过载浓缩成一个单一的放大倍数，便于工程计算。

在本工具中 K_A 的角色：
```
# 在 core/spline/calculator.py:calculate_spline_fit 入口
torque_design_nm = torque_required_nm * ka         [N·m → N·m]
axial_design_n   = axial_required_n * ka           [N → N]
```
这两个放大后的设计载荷被传入场景 A（齿面承压）与场景 B（过盈防滑 / 屈服），两个场景都受 K_A 影响。

**为什么 K_A 与 k_α 要区分**：

这是**两个独立来源的放大系数**，不能合并：
- **K_A**：外部工况的冲击 / 过载（驱动类型 → 负载类型）。单位：无量纲。
- **k_α**：花键**内部**的载荷分布不均（齿向偏载 + 齿面接触不均的合成）。单位：无量纲。

在齿面压力公式里两者相乘出现：`p_flank = 2·(T_req·K_A)·k_α / (z·h_w·d_m·L)`。工具把 K_A 放在入口预乘、把 k_α 放在几何章节 —— 两者分开填、分别调整。把任一项漏填当作 1.0 会让 p_flank 偏低 20~50%。

**典型值 / 场景**（DIN 6892 风格的简化分类）：

- **1.00 ~ 1.25**：均匀负载 + 均匀驱动。例：电机驱动 + 恒定载荷（风扇、泵）。本工具默认 1.25。
- **1.25 ~ 1.50**：轻度冲击 / 启停。例：电机驱动 + 带负载波动的输送带、机床主轴。
- **1.50 ~ 1.75**：中等冲击。例：内燃机驱动、频繁启停的卷扬机、锤击负载。
- **1.75 ~ 2.25**：严重冲击。例：轧机 / 破碎机 / 冲压机 / 矿山设备。
- **> 2.25**：极端冲击或非稳态剧烈工况，超过 DIN 6892 标准建议范围时需结合实测载荷谱。

**常见误用**：

- **已经填了"峰值扭矩"又选 K_A = 1.5**：双倍放大。K_A 的前提是名义扭矩。填峰值后应 K_A = 1.0。
- **混淆 K_A 与 k_α**：见上节。
- **驱动侧 / 负载侧各一个 K_A 相乘**：某些标准里 K_A 是"驱动 × 负载"查表，本工具已经假设用户已经完成查表得到单一合成值；不要再分别乘。
- **取值偏低（K_A = 1.0）做批准依据**：除非确实是恒功率 / 恒载荷的稳态工况（很少见），否则留一档是合理工程裕度。

**出处**：DIN 6892:2012（花键连接承载能力）、DIN 3990-1（齿轮 K_A 取值参考，花键常借用齿轮经验）。
