# 载荷分布系数（k_α）

**一句话**：把花键内部齿向偏载与齿面接触不均两种"载荷分布不均"因素合成的一个放大系数；在齿面压力公式里直接乘到设计扭矩上，默认取 1.3（保守上限）。

**怎么理解**：

理想花键假设所有 z 颗齿都均匀承担扭矩、每颗齿在 L 长度上压力均匀分布。实际装配后：

1. **齿向偏载（K_2）**：轴和轮毂不完全对中（制造误差、弹性变形、轴挠曲），导致 L 长度上一端压力高、另一端压力低；极端情况下只有一端压力较高的几 mm 真正受力。
2. **齿间载荷不均（K_1）**：z 颗齿的加工精度略有差异，啮合初期只有"最紧的几颗齿"受力；随着载荷增加、齿逐渐磨合，载荷慢慢分散到所有齿 —— 但传递峰值时仍非均匀分配。
3. **齿面接触不均（K_3）**：单颗齿沿 h_w 径向上的压力也不均（渐开线齿形 + 加工偏差）。

DIN 6892:2012 把这三个效应分别用 K_1 / K_2 / K_3 系数描述，最终合成的载荷放大系数为 `K_comp = K_1 · K_2 · K_3`。**本工具把三者合成为单一 `k_α`**，不做逐一分解 —— 取保守上限作为简化处理。

**在齿面压力公式里的位置**：

```
p_flank = 2 · T_design · k_α / (z · h_w · d_m · L)
         │   │       │
         │   │       └── 载荷分布系数，本字段
         │   └─────────── K_A 预乘后的设计扭矩
         └───────────────── 因子 2 来自 T = F·r, F 是切向力

单位：[T_design: N·mm, k_α: 无量纲, z: 无量纲, h_w: mm, d_m: mm, L: mm → p_flank: MPa]
注意: calculator 用 N·m 输入，内部转 N·mm (× 1000)；若手算请先换算单位。
```

注意：`core/spline/calculator.py:_calculate_scenario_a` 里只显式出现 k_α（不是 K_A），因为 K_A 已经在入口 `calculate_spline_fit:222` 把 T 放大了：`torque_design_nm = torque_required_nm * ka`。k_α 只负责花键**内部**的载荷分布，不与 K_A 重复。

**典型值 / 场景**：

- **k_α ≈ 1.0 ~ 1.15**：理想过盈固定花键（无间隙、短 L、精密对中）。实际很难达到，需要精密加工 + 严格装配。
- **k_α ≈ 1.2 ~ 1.3**：常规过盈固定花键。**本工具默认 1.3，偏保守一档**。适合大多数工业传动。
- **k_α ≈ 1.3 ~ 1.5**：带少量间隙的固定花键，或 L / d_B 比较大（轴向长啮合 → 齿向偏载加剧）。
- **k_α ≈ 1.5 ~ 2.0**：滑移花键（运转中可轴向滑动）、或高频振动场合。
- **k_α > 2.0**：极端工况 / 严重偏心 / 齿面磨损后期；正常设计不取这么大，应重新审视结构。

**常见误用**：

- **把 k_α 与 K_A 合成一个系数填**：两者独立，一个是外部冲击、一个是内部分布。合并会让人在某场景切换时误改了不该改的。
- **误以为 `k_α = 1.0` 是"精密一档"的默认**：DIN 6892 原始框架里 K_1、K_2、K_3 对绝大多数实际工况都 > 1.0（没有理想均布的花键）。取 k_α = 1.0 意味着相信所有三个系数同时等于 1，工程上不现实。
- **基于"滑移花键"场景取 1.0**：反过来 —— 滑移花键载荷分布**更差**（有间隙），应该取 1.5~2.0。

**出处**：DIN 6892:2012（K_1 · K_2 · K_3 系数框架）；本工具合成约定见 `core/spline/calculator.py:51-55` 注释。

> Cannot verify against original DIN standard —— DIN 6892 对 K_1 / K_2 / K_3 的精确计算表格未逐条比对原文。本工具取单一 k_α 保守上限，不替代 DIN 6892 完整分解计算。
