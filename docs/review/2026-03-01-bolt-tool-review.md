# 螺栓工具基础问题审查报告

**日期：** 2026-03-01
**审查范围：** `app/` UI 层 + `core/bolt/calculator.py` 计算内核
**原则：** 校核结果是工程可用的，不是玩具。每条问题都从"是否影响可靠性或可用性"角度评级。

---

## 一、UI 交互问题

### 1.1 窗口无有效最小尺寸约束 【影响：可用性】

- **位置：** `app/ui/main_window.py:35`，`app/ui/pages/bolt_page.py:458`
- **现象：**
  - 主窗口初始 `resize(1400, 860)`，但无 `setMinimumSize()` 限制
  - 左侧模块栏固定宽度 270px，章节导航栏固定宽度 280px
  - 缩小窗口时，内容区域被挤压至极小甚至不可用
- **影响：** 在较小屏幕或分屏使用时，输入区域可能消失

### 1.2 输入字段缺乏有效引导 【影响：误操作风险】

- **位置：** `app/ui/pages/bolt_page.py:578`
- **现象：**
  - 所有数值输入框的 placeholder 统一为 `"请输入数值"`，没有值域范围提示
  - 字段说明（hint）只在**聚焦/悬停时**才显示在底部 info_label，用户容易错过
  - 没有"必填"与"选填"的视觉区分（内部校验有区分，但界面上看不出来）
  - 输入错误（非数字）只在点击"执行校核"后才弹框报错，没有实时反馈
- **工程场景：** 用户填错 `alpha_A=1` 或 `mu_thread=0.12`（应为 0.12，但填成 12）时，计算前毫无提示

### 1.3 初始状态 badge 语义混淆 【影响：可读性】

- **位置：** `app/ui/pages/bolt_page.py:483`
- **现象：**
  - `overall_badge` 初始文本"等待计算"，但使用的是 `FailBadge` 样式（红底深红字）
  - 新用户打开工具，看到红色"等待计算"，会误以为已有校核失败

### 1.4 章节导航宽度固定，不支持缩放 【影响：可用性】

- **位置：** `app/ui/pages/bolt_page.py:458`：`nav_card.setFixedWidth(280)`
- **现象：** 左侧章节导航固定 280px，章节名称较长时（如"14.5 Joint Elements"）会截断显示
- **建议：** 改为最小宽度约束 `setMinimumWidth`，允许拖动分割

### 1.5 示意图（clamping_diagram）表现不足 【影响：工程沟通质量】

- **位置：** `app/ui/widgets/clamping_diagram.py`
- **现象：**
  - 力箭头位置和大小均为硬编码，不随 FM/FA/FK 数值按比例变化
  - 没有 VDI 2230 的"螺栓力-变形特性图"（Verspannungsschaubild），无法直观传达"力分配"概念
  - 图中只有三个符号（FM/FA/FK），缺少载荷进入点、弹性区域等关键信息
  - FK 为 0 时（极端工况），图上无法区分"接近开口"状态

## 二、计算逻辑问题（工程准确性）

### 2.1 `FA_perm` 公式非 VDI 2230 标准 【影响：结果可靠性 ⚠️ 重要】

- **位置：** `core/bolt/calculator.py:177`

```python
f_a_perm = 0.1 * rp02 * geometry["As"] / phi_n
```

- **问题：** VDI 2230 没有定义形如 `0.1 * Rp0.2 * As / phi_n` 的"允许轴向载荷"。这个公式的 `0.1` 系数来源不明，会导致：
  - 高强度螺栓（Rp0.2 大）时允许值虚高
  - 软质材料螺栓时允许值偏低
  - 结果显示"附加载荷能力校核：通过"，但依据不是标准中的规则
- **正确做法：** VDI 2230 对轴向载荷的控制是通过预紧力需求（R1）和应力校核（R4/R5）实现的，不单独定义 FA_perm。此检查项应明确标注"工程估算（非VDI标准项）"，或改为基于明确公式的规则。

### 2.2 警告信息为英文，在中文界面中出现 【影响：可读性】

- **位置：** `core/bolt/calculator.py:182-186`

```python
warnings.append("phi_n >= 1.0, axial external load almost fully enters bolt; check stiffness model and n.")
warnings.append("High assembly utilization (>0.95). Verify friction scatter and process capability.")
```

- **现象：** 这些警告最终显示在"消息与建议"区（`bolt_page.py:839`），但语言不一致

### 2.3 缺少 VDI 2230 中的其他基础校核项 【影响：覆盖率】

当前仅有 4 个校核项，以下 VDI 2230 常规校核项缺失（首版注明了"不覆盖"，但需要在界面上说清楚边界）：

| 缺失校核 | VDI 2230 条款 | 工程意义 |
|---------|--------------|---------|
| 支承面压强 p ≤ p_allow | R5 | 防止螺母/垫片下陷 |
| 螺纹脱扣强度 | R6 | 防止内螺纹滑丝 |
| 疲劳校核（应力幅值） | R7 | 动载工况必须项 |
| 间隙/密封安全系数 | R3 | 有密封要求时 |

- **建议：** 在结果页或界面上明确标注"本版本覆盖范围：装配强度、服役强度、残余夹紧力（非完整 VDI 2230）"

### 2.4 输入参数校验不完整 【影响：物理合理性】

- **位置：** `core/bolt/calculator.py`
- **现象：** 除正值检查外，以下物理约束未检验：
  - `alpha_A ≥ 1`（散差系数必须 ≥ 1，否则 FMmax < FMmin 违反物理意义）
  - `0 < mu_thread ≤ 1` 和 `0 < mu_bearing ≤ 1`（摩擦系数越界）
  - `0 < utilization ≤ 1`（利用系数超过 1 没有物理意义）
  - `n ≤ 1`（载荷导入系数应 ≤ 1，当前只检查 >0）

### 2.5 结果页没有利用率（ratio）展示 【影响：工程判断】

- **位置：** `app/ui/pages/bolt_page.py:827-835`
- **现象：** 当前只显示绝对值，如"装配等效应力: 580 MPa (允许 576 MPa)"，工程师更习惯看利用率"100.7%"，方便评估裕量大小

## 三、架构问题（面向后续扩展）

### 3.1 没有 `core/common/` 共用模块

- 路线图中规划了 `core/common/`（单位换算、材料库），但尚未创建
- 当前所有公式直接写在 `calculator.py` 中，后续其他模块无法复用

### 3.2 没有标准数据库（`data/standards/`）

- 无 ISO 公制螺纹规格表（d, p, As 标准值）
- 无材料强度等级表（8.8/10.9/12.9 的 Rp0.2 查表）
- 用户每次都要手动查手册填入，容易出错

### 3.3 占位模块缺乏描述

- **位置：** `app/ui/pages/placeholder_page.py`
- **现象：** 占位页只写"模块框架已预留，当前版本未实现"，没有说明该模块的计算范围、参考标准、预计实现顺序

### 3.4 无计算会话保存/加载

- 当前只有"加载样例"和"导出报告"，不能保存用户输入的一套参数并下次继续使用
- 缺少项目管理层（工程师常需要"昨天的那个 M10 x 50 工况"）
