# 输入条件操作栏重构设计

## 目标
统一三个工程模块页面顶部操作栏的结构与命名：左侧负责“输入条件”的保存、加载以及计算相关操作；右侧统一放置“测试案例”按钮，并把原“加载样例”文案统一改为“测试案例”。

## 范围
- 覆盖页面：`app/ui/pages/bolt_page.py`、`app/ui/pages/interference_fit_page.py`、`app/ui/pages/hertz_contact_page.py`
- 共享能力：项目内 `saved_inputs/` 目录的 JSON 保存/加载
- 不改章节导航结构，不改计算逻辑，不改样例文件内容

## 交互设计
- 左侧操作区：`保存输入条件`、`加载输入条件`、`执行校核`、`清空参数`、`导出结果说明`
- 右侧操作区：`测试案例 1`、`测试案例 2`
- 保存/加载使用文件选择框，默认定位到项目内 `saved_inputs/` 目录
- 输入条件以 JSON 文件形式持久化，关闭软件后仍可在下次启动时继续加载

## 技术设计
- 新增共享输入条件 I/O helper，负责：
  - 解析项目根目录下默认保存目录
  - 通过 `QFileDialog` 获取保存/加载路径
  - 读写 UTF-8 JSON，并确保目录存在
- 页面层复用现有 payload 构建与表单赋值逻辑：
  - 保存时使用各页面已有 `_build_payload()` 结果
  - 加载时复用各页面现有“按 mapping 回填控件”的逻辑，避免重复引入第二套字段映射规则
- `BaseChapterPage` 增加左右分区操作栏 API，过盈配合与赫兹接触直接复用；螺栓页保留现有页面结构，只把操作栏改成相同的左右分区布局

## 错误处理
- 保存前若输入无法构建 payload，直接复用现有输入错误提示
- 加载文件不存在、JSON 非法或字段不兼容时弹出消息框
- 加载后同步刷新页面已有的联动项（材料、模式、图示、层级显示）

## 测试策略
- 先为共享 JSON helper 写失败测试，验证默认目录和保存/读取行为
- 现有计算测试保持不变，新增 helper 测试确保基础持久化逻辑可回归
- 完成后运行新增测试与现有核心测试，确认没有破坏已实现模块
