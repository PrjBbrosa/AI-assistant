# 蜗轮蜗杆模块打磨设计（DIN 3975 几何 + Manual 风格重构）

## 1. 目标
在现有蜗轮和蜗杆模块首版基础上，继续按附件 manual 的界面组织和执行逻辑打磨页面，但严格区分“必要输入”和“自动计算尺寸”，避免将派生结果继续作为普通输入项造成混淆。

本轮打磨要同时解决四类问题：
- 图形与曲线页面点击后闪退
- 几何参数中必要输入与派生尺寸混在一起
- 蜗杆相关尺寸和蜗轮相关尺寸没有分列展示
- 图形示意不够像蜗杆/蜗轮副，工程语义不明确

## 2. 设计原则
### 2.1 manual 对齐原则
- 页面顺序、说明方式、术语组织尽量贴近 manual
- 界面内不使用 `chapter` 命名，统一使用“步骤”“结果”“说明”等中文表述
- 图示说明要具有工程图语义，而不是泛用机械草图

### 2.2 输入边界原则
- 只把真正驱动几何链路的量作为默认输入
- 所有可由核心输入推导出的尺寸统一放入只读展示区
- 允许保留一个“高级参数”折叠区，用于少数工程上确实需要覆盖的经验参数或派生量
- 全面贴 manual 不等于把所有尺寸都暴露为可编辑输入框

### 2.3 图形稳定性原则
- 所有自绘组件必须在真实绘制路径下可安全执行
- 图形页必须支持滚动，不能因内容高度导致布局或绘图异常
- 任何绘图失败都不能让整个模块崩溃

## 3. 页面结构
步骤顺序保持为：
1. `基本设置`
2. `几何参数`
3. `材料与配对`
4. `公差与回差`
5. `工况与润滑`
6. `图形与曲线`
7. `Load Capacity`
8. `结果与报告`

### 3.1 顶部操作区
保持现有统一风格：
- 左侧：`保存输入条件`、`加载输入条件`、`执行计算`、`清空参数`、`导出结果说明`
- 右侧：`测试案例 1`、`测试案例 2`

### 3.2 几何参数页布局
几何参数页重构为四个区域：
- 左列：`蜗杆参数`
- 右列：`蜗轮参数`
- 中间或下方：`啮合与装配`
- 下半区两列只读：`蜗杆自动计算尺寸`、`蜗轮自动计算尺寸`

页面内部必须使用滚动区域，保证在较小窗口下仍可完整浏览。

## 4. 参数分层
### 4.1 基础输入
基础输入是默认显示、直接驱动 DIN 3975 几何链路的量，建议包含：
- 蜗杆：`z1`、模数 `m`、直径系数 `q`、导程角 `gamma`、齿宽或螺纹长度、旋向
- 蜗轮：`z2`、齿宽
- 啮合装配：实际中心距、安装形式
- 工况：输入功率、输入转速、使用系数、润滑方式
- 材料：蜗杆材料、蜗轮材料

### 4.2 装配与公差输入
这一层保持为输入，因为工程上会直接指定：
- 齿厚偏差或公差带
- 中心距偏差
- 法向回差
- 目标侧隙
- 安装修正项
- manual 风格的保留记录项

### 4.3 自动计算尺寸
所有可由基础输入推导出的尺寸统一进入只读结果区，不在主输入区重复出现。

蜗杆自动计算尺寸建议包含：
- 分度圆直径
- 顶圆直径
- 根圆直径
- 导程
- 轴向节距
- 圆周速度

蜗轮自动计算尺寸建议包含：
- 分度圆直径
- 顶圆直径
- 根圆直径
- 齿高
- 圆周速度

啮合结果建议包含：
- 传动比
- 理论中心距
- 中心距偏差
- 输出转速
- 输出扭矩

### 4.4 高级参数
高级参数默认折叠，仅允许覆盖少数需要工程调整的量：
- 摩擦系数或经验修正
- 散热经验系数
- 预留给 `DIN 3996` 的方法参数
- 少量需要人工覆盖的派生量

## 5. 图形与曲线
### 5.1 几何示意图
几何图仍可保持占位图策略，但必须明确表现为蜗杆和蜗轮副：
- 显示两根轴线
- 显示蜗杆螺旋特征
- 显示蜗轮分度圆关系
- 标出中心距、导程角、旋向和主要尺寸说明
- 右侧附工程说明块

### 5.2 公差与回差示意图
公差图不要求实时联动，但必须明确对应蜗轮副概念：
- 齿厚偏差
- 公差带
- 法向回差
- 中心距偏差
- 安装修正或装配余量说明

### 5.3 性能曲线
性能曲线保持真实计算与绘图：
- 效率曲线
- 损失功率曲线
- 热功率相关曲线
- 当前工况点
- 图例、横轴说明、结果摘要

曲线仍然在点击 `执行计算` 后刷新。

## 6. 闪退根因与修复要求
已确认的根因：
- 图形组件在 `paintEvent` 中使用了错误的 Qt 枚举 `Qt.AlignmentFlag.TextWordWrap`
- 真实绘图时抛出异常，导致 `QPainter` 未正常结束
- 页面进入图形步骤时触发重绘，因此表现为闪退

修复要求：
- 改用正确的文本换行标志
- 自绘组件保证异常情况下也不会留下活动 painter
- 图形页改为可滚动布局
- 修复后必须通过真实 `show()` / `grab()` 路径验证，而不仅是实例化 smoke test

## 7. 数据流
### 7.1 即时派生结果
只读尺寸区随基础输入变化即时更新，不必等待点击 `执行计算`。

### 7.2 执行计算结果
点击 `执行计算` 后统一刷新：
- 性能曲线
- 结果摘要
- Load Capacity 骨架状态

### 7.3 输入条件保存
保存内容继续覆盖：
- 基础输入
- 公差与回差输入
- 高级参数
- 方法选择与页面状态

## 8. 测试要求
本轮打磨至少新增或加强以下测试：
- 图形页切换到“图形与曲线”后 `show()` / `grab()` 不崩
- 主窗口进入蜗轮模块并切换到图形页不崩
- 几何参数页存在 `蜗杆参数`、`蜗轮参数`、`蜗杆自动计算尺寸`、`蜗轮自动计算尺寸`
- 修改基础输入后，只读尺寸区会立即更新
- 执行计算后，性能曲线仍然输出真实三组曲线和当前点

## 9. 与 DIN 3996 的边界
本轮仍不开始 `DIN 3996` 真正校核，只优化以下内容：
- `Load Capacity` 页结构
- 方法保留
- 输入保存能力
- 未开始状态提示

完成本轮后，仍需再次明确提醒用户：
`是否开始 DIN 3996 校核部分？`
