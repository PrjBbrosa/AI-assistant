# 个人版「eAssistant 风格」平台路线图（先做螺栓模块）

## 1. 目标定义
- 目标：构建一个可扩展的“机械连接计算平台”个人版，架构上预留多模块能力，首期只交付螺栓校核模块。
- 原则：做“功能等效与工程可用”，不复制对方品牌、UI 资源或私有实现细节。

## 2. 三种实施路径

### 路径 A（推荐）：`Python + PySide6 桌面应用`
- 优点：本地运行体验好，适配 PyCharm 开发和后续 `.exe` 打包。
- 缺点：跨平台 UI 细节需额外测试。

### 路径 B：`Python + Tkinter`
- 优点：依赖少。
- 缺点：视觉与交互能力较弱，不利于后续复杂模块。

### 路径 C：`Python + Web(本地服务)`
- 优点：界面灵活。
- 缺点：部署链路更复杂，不符合当前“纯本地桌面优先”。

## 3. 推荐方案与边界
- 选用路径 A（PySide6 桌面）。
- 首期只完成“螺栓模块”端到端闭环，其它模块只做入口和占位。
- 标准边界：以 VDI 2230 核心链路为主，完整标准全场景逐步补齐。

## 4. 功能分层（平台级）

### 4.1 平台层（先搭框架）
- 模块导航与项目管理
- 计算任务管理（输入、版本、结果、导出）
- 材料/紧固件数据库（先本地 JSON，后续数据库）
- 公式与审计日志（记录每次计算参数与版本）

### 4.2 计算模块层（先预留）
- 螺栓连接（首期实现）
- 轴连接（占位）
- 轴承（占位）
- 蜗轮（占位）
- 弹簧（占位）
- 材料与标准库（占位）

### 4.3 通用能力层
- 单位系统与换算
- 标准参数库（ISO 螺纹、材料等级）
- 报告生成（PDF/HTML）

## 5. 首期（螺栓模块）能力清单
- 输入建模：几何、材料、摩擦、刚度、载荷、装配参数
- 核心计算：`FMmin/FMmax`、载荷分配、扭矩、装配/服役校核
- 结果解释：分项通过/不通过、关键敏感参数提示
- 报告导出：JSON + 可读报告（首期 HTML）

## 6. 推荐工程结构

```text
bolt-tightening-platform/
  app/                   # PySide6 桌面应用
    ui/                  # 主题、窗口、页面
  core/
    bolt/                # 螺栓模块（VDI 2230）
    common/              # 通用能力（后续）
  data/
    standards/           # 标准参数表（螺纹、材料）
  scripts/
    build_exe.bat
  docs/
    plans/
    specs/
```

## 7. 迭代里程碑（可执行）

- 完成 PySide6 框架与模块导航
- 把当前 `src/vdi2230_tool.py` 拆入 `core/bolt`

### M2（1 周）：参数库与报告
- 接入 ISO 螺纹/材料等级基础库

### M3（持续）：螺栓模块增强
- 增加刚度自动建模
- 扩展疲劳校核、更多工况模板
- 增加参数敏感性分析

## 8. 风险与控制
- 风险：标准细节覆盖不足导致“看似通过”但边界工况偏差大。
- 控制：
  - 每个公式给出来源与实现注记
  - 每次发布保留基准算例回归
  - 明确输出“适用范围与未覆盖项”
