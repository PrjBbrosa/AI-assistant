## 二、基础设施层（所有模块共用）

在加入第二个计算模块前，需要建立以下共用层，否则每个模块都要重复造轮子。

### 2.1 `core/common/` — 共用计算工具

```
core/common/
├── __init__.py
├── units.py          # 单位换算（N↔kN, mm↔m, MPa↔N/mm²）
├── materials.py      # 材料数据库接口（查表 + 手动输入）
└── validators.py     # 通用物理约束校验（正数、范围、单位检查）
```

**优先实现：**
- `validators.py`：封装 `_positive()`、范围检查等，供所有模块计算器使用
- `materials.py`：ISO 强度等级查表（4.6/6.8/8.8/10.9/12.9 的 Rp0.2/Rm）

### 2.2 `data/standards/` — 标准数据文件

```
data/standards/
├── iso_metric_threads.json    # ISO 公制螺纹规格（d, p, As, d2, d3 标准值）
├── material_grades.json       # 螺栓材料强度等级表
└── bearing_standards.json     # 轴承标准数据（后续用）
```

**螺纹表示例（优先）：**
```json
[
  { "d": 8,  "p": 1.25, "As": 36.6,  "d2": 7.188, "d3": 6.647 },
  { "d": 10, "p": 1.5,  "As": 58.0,  "d2": 9.026, "d3": 8.376 },
  { "d": 12, "p": 1.75, "As": 84.3,  "d2": 10.863, "d3": 10.106 },
  { "d": 16, "p": 2.0,  "As": 157.0, "d2": 14.701, "d3": 13.835 }
]
```

**作用：** 用户选择 M10×1.5 后，As/d2/d3 自动填入，减少手工查手册。

### 2.3 `app/ui/base_module_page.py` — 模块页面基类

所有计算模块页面共用的 UI 骨架：

```python
class BaseModulePage(QWidget):
    """所有计算模块页面的基类。"""
    # 标准章节导航（左侧 QListWidget + 右侧 QStackedWidget）
    # 标准操作栏（加载样例、执行校核、清空、导出）
    # 标准底部状态栏（总体通过/不通过 badge + 提示文字）
    # 通用 eventFilter（字段聚焦显示说明）
```

**价值：** 第二个模块不需要复制 `bolt_page.py` 的 UI 骨架，只需继承并填入自己的 `CHAPTERS` 和计算函数。

## 三、各模块扩展计划

### 模块 M2：轴连接（优先级：第二）

**参考标准：** DIN 6885（键连接）、DIN 7190（过盈配合）、ISO 286（公差与配合）

**计算范围（首版）：**
- 平键连接：键的挤压应力、剪切应力校核
- 过盈配合：装配力、传递扭矩能力、应力校核（拉梅方程）

**章节组织（参考 eAssistant）：**
1. 连接类型与工况（键/花键/过盈）
2. 几何参数（轴径、毂径、键槽尺寸）
3. 材料属性（轴/毂屈服强度）
4. 载荷输入（传递扭矩、轴向力）
5. 结果：校核结论、安全系数、建议

**前置依赖：**
- `core/common/materials.py`（材料查表）
- `BaseModulePage`（UI 骨架）

---

### 模块 M3：轴承（优先级：第三）

**参考标准：** ISO 281（额定疲劳寿命）、ISO 76（静额定载荷）

**计算范围（首版）：**
- 径向球轴承/滚子轴承
- 当量动载荷 P = X·Fr + Y·Fa
- 额定寿命 L10h = (C/P)^p · (10^6/60n)

**章节组织：**
1. 轴承类型与尺寸（d/D/B，C/C0 额定值）
2. 载荷（径向载荷 Fr，轴向载荷 Fa）
3. 运行条件（转速 n，工作温度）
4. 寿命目标（L10h 设计要求）
5. 结果：当量载荷、计算寿命、通过/不通过

**前置依赖：**
- `data/standards/bearing_standards.json`（标准型号数据）
- `BaseModulePage`

---

### 模块 M4：弹簧（优先级：第四）

**参考标准：** DIN 2089/2090（螺旋弹簧）、DIN 2093（碟形弹簧）

**计算范围（首版）：**
- 圆柱螺旋压缩弹簧
- 弹簧刚度、自由高度、应力校核、共振频率

**章节组织：**
1. 材料（弹簧钢牌号、G 模量）
2. 几何（线径 d、中径 D、有效圈数 n）
3. 载荷（工作力 F1/F2，工作行程）
4. 结果：刚度、应力、疲劳安全系数

---

### 模块 M5：蜗轮（优先级：第五）

**参考标准：** DIN 3996（蜗轮蜗杆传动设计）、ISO 14521

**计算范围（首版）：**
- 蜗杆几何（模数、头数、导程角）
- 传动效率、热平衡
- 蜗轮齿面接触与弯曲强度校核

**章节组织：**
1. 几何参数（模数 m, 头数 z1, 齿数 z2, 导程角）
2. 材料（蜗杆/蜗轮材料）
3. 载荷与润滑
4. 热功率校核
5. 强度结果

---

### 模块 M6：材料与标准库

**性质：** 辅助查询工具，非计算模块

**功能（首版）：**
- ISO 公制螺纹规格查询（M3-M100）
- 螺栓强度等级参数查询（4.6～12.9）
- 常用轴承型号 C/C0 查询

**后续：**
- 材料牌号库（钢、铸铁、铝合金、工程塑料）
- 标准表查询（公差配合、粗糙度、表面处理）

## 五、架构约束与原则

1. **计算内核与 UI 解耦：** `core/` 永远不依赖 `app/`。每个模块计算器可以独立 CLI 调用。

2. **公式必须有来源注记：** 每个公式实现旁边注明标准条款号，方便审查。

3. **每个模块必须有至少 2 个验证样例：** 一个失败工况、一个通过工况，对齐计算结果。

4. **版本向前兼容：** 同一模块的输入 JSON schema 变更，旧样例文件必须能继续加载（可接受新字段可选）。

5. **不过度抽象：** BaseModulePage 只抽象真正共用的 UI 骨架，不强求所有模块完全一致。如果某个模块的 UI 差异大，允许直接继承 QWidget 自己实现。
