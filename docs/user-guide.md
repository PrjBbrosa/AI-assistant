# Local Engineering Assistant 新手使用说明

本工具是一个本地桌面版机械计算助手，适合做方案比选、快速校核、结果整理和报告导出。

先看结论：

| 模块 | 适合做什么 | 常用操作 | 当前状态 |
|------|------------|----------|----------|
| 螺栓连接 | VDI 2230 核心链路校核 | 保存/加载输入、测试案例、执行校核、导出报告、查看校核指南 | `已可用`；但还不是完整 VDI 2230，全偏心/弯矩、螺纹脱扣、完整疲劳谱等未覆盖 |
| 过盈配合 | 圆柱面过盈、装配过程、压入力曲线、Fretting 风险 | 保存/加载输入、测试案例、执行校核、导出报告 | `已可用`；但 `service temperature / speed / centrifugal force / stepped geometry` 仍未并入主模型，ISO 286 仅支持精选子集 |
| 花键连接校核 | 花键齿面承压预校核 + 光滑段圆柱过盈 | 计算、导出报告 | `部分完成`；场景 A 目前只是简化预校核，不能当作正式 `DIN 5480 / DIN 6892` 签发结果 |
| 蜗轮蜗杆设计 | 蜗杆/蜗轮几何、基础性能、Method B 最小子集 | 保存/加载输入、测试案例、执行计算、导出报告 | `部分完成`；还不是完整 `DIN 3996 / ISO/TS 14521` |
| 赫兹应力 | 线接触/点接触的快速接触应力估算 | 保存/加载输入、测试案例、执行校核、导出报告 | `首版可用`；更适合快速估算，不是完整轴承模块 |
| 材料与标准库 | 未来材料库/标准库入口 | 无 | `未完成`；当前只是占位页 |

## 1. 启动工具

推荐在 Windows 或 PyCharm 下使用。

如果项目里还没有装好依赖，在项目根目录执行：

```powershell
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe app\main.py
```

如果 `.venv` 已经存在，直接执行最后两行即可。

PyCharm 用户也可以直接把项目解释器指向 `.venv`，然后运行 [app/main.py](/D:/AI-assistant/app/main.py)。

## 2. 第一次使用建议

建议按下面顺序上手：

1. 打开软件后，先从左侧选择一个模块。
2. 第一次使用时，优先点页面上的“测试案例 1”或“测试案例 2”。
3. 先运行一次，再回头看各章节字段，理解每个参数大概在控制什么。
4. 确认单位后，再把测试值逐项替换成你的实际工况。
5. 点击“执行校核”或“执行计算”。
6. 先看页面顶部的总体结论，再看各章节的分项结果、提示和警告。
7. 结果需要留档时，再点“导出结果说明”或“导出报告”。
8. 如果当前模块支持“保存输入条件”，建议把本次工况存成一个 JSON，方便下次继续修改。

## 3. 页面上这些按钮是什么意思

| 按钮 | 作用 |
|------|------|
| 保存输入条件 | 把当前表单保存成 JSON，通常保存在项目里的 `saved_inputs/` 目录 |
| 加载输入条件 | 读取之前保存的 JSON，恢复表单 |
| 测试案例 1 / 2 | 加载仓库自带样例，适合先跑通流程 |
| 执行校核 / 执行计算 / 计算 | 根据当前输入开始计算 |
| 清空参数 | 把当前表单恢复到初始状态 |
| 导出结果说明 / 导出报告 | 导出 PDF、TXT，部分模块还能导出 DOCX 或富格式 PDF |
| 校核指南 | 目前螺栓模块独有，适合第一次看 VDI 2230 的新手 |

说明：

- 花键连接校核模块目前没有“保存输入条件 / 加载输入条件 / 测试案例”按钮。
- 不同模块的导出格式略有差异，但都能导出可读文字结果。

## 4. 新手最推荐的使用顺序

### 4.1 螺栓连接

最适合第一次上手。

推荐流程：

1. 打开“螺栓连接”。
2. 先点“测试案例 1”。
3. 点一次“校核指南”，先理解 `FMmin / FMmax / R3 / R4 / R5` 是什么。
4. 切回“输入步骤”，按章节从上到下检查参数。
5. 点击“执行校核”。
6. 看顶部总体结论，再看“校核链路”页定位是哪一步没通过。

### 4.2 过盈配合

推荐第二个上手，因为结果区最完整。

推荐流程：

1. 打开“过盈配合”。
2. 先点“测试案例 1”。
3. 先确定“过盈来源模式”。
4. 再确定“装配模式”是 `manual_only`、`shrink_fit` 还是 `force_fit`。
5. 执行校核后，先看总体结论，再看压入力曲线和 Fretting 风险段。

### 4.3 赫兹应力

适合做最短路径试算。

推荐流程：

1. 先确定是“线接触”还是“点接触”。
2. 填半径、材料、法向载荷。
3. 执行校核后，直接看 `p0`、安全系数和曲线。

### 4.4 蜗轮蜗杆设计

适合已有基础参数的人使用。

推荐流程：

1. 先把几何参数和材料参数填完整。
2. 如果要看负载能力，再启用 `Load Capacity`。
3. 运行后先看几何与基础性能，再看齿面应力、齿根应力和扭矩波动。

### 4.5 花键连接校核

这个模块目前更适合“会看边界说明的人”。

推荐流程：

1. 优先使用图纸或公开目录尺寸，不要优先用“近似推导”模式。
2. 把它当作预校核，不要直接拿来做正式签发。
3. 如果用了联合模式，再额外核对场景 B 的圆柱过盈结果。

## 5. 明确未完成或仍在扩展的部分

下面这些地方要特别留意：

### 5.1 明确未完成

- `材料与标准库`：当前只有入口，没有实际功能。

### 5.2 已上线，但仍是“部分完成”

- `花键连接校核`
  - 场景 A 还是简化预校核。
  - 还没做到完整公差/变位解析、完整分载因子链、齿根/剪切等失效模式。
  - 还不适合直接作为正式 `DIN 5480 / DIN 6892` 工程结论。
- `蜗轮蜗杆设计`
  - 当前是 `DIN 3975 + Method B 最小子集`。
  - 还没做到完整 `DIN 3996 / ISO/TS 14521`。
- `过盈配合`
  - 还没把 `service temperature / speed / centrifugal force / stepped geometry` 纳入主模型。
  - ISO 286 也不是完整公差库，只做了精选子集。
- `螺栓连接`
  - 核心链路可用，但偏心/弯矩、螺纹脱扣、完整疲劳谱等还没做全。
- `赫兹应力`
  - 当前是快速计算模块，不是完整轴承分析模块。

## 6. 使用时最容易踩的坑

- 单位要统一：本工具大多使用 `mm / N / MPa / N·m`。
- 不清楚字段含义时，不要直接填正式工况，先加载测试案例跑一遍。
- 材料下拉框切到“自定义”后，要自己确认 `E`、`nu`、许用应力等字段是否也一起改对。
- 看见“通过”也不代表已经覆盖了全部标准条款，仍要结合本页的“当前状态”和“未完成项”判断。

## 7. 给新手的简短建议

如果你是第一次用，最稳妥的顺序是：

1. 先用“螺栓连接”熟悉页面节奏。
2. 再用“过盈配合”理解结果页和曲线。
3. 需要快速验证接触压力时，用“赫兹应力”。
4. “花键连接校核”和“蜗轮蜗杆设计”先按预校核/子集模块理解，不要默认它们已经是完整标准版。

如果你要做批处理或外部集成，可以再看保留的 CLI 入口：[src/vdi2230_tool.py](/D:/AI-assistant/src/vdi2230_tool.py)。
